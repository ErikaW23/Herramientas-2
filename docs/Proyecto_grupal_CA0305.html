Ejercicio 1: fundamentos del broadcasting en tensores

Implemente una función broadcast_ops(X, b, w) que realice la operación Y = (X + b) * w, donde X es una matriz de tamaño (N, D), b es un vector de tamaño (D,) y w es un vector de tamaño (N,).

Requerimientos
1- Retornar el tensor resultante Y de tamaño (N, D).
2- Usar operaciones de la librería Numpy.
3- No utilizar bucles for explícitos (hacer el código vectorizado).

Explicación de la fundamentación matemática

En cuanto a principios matemáticos, esto no se trata de una multiplicación de matrices como en la teoría, si no que se utiliza la dimensión de los vectores b y w con el objetivo de multiplicar matrices elemento por elemento.

Es importante mencionar otro principio matemático, y es que las dimensiones de las matrices y vectores deben coincidir. Por ejemplo, si X tiene dimensiones (N, D), entonces para sumar por cada elemento debemos hacer que el vector que se suma entrada por entrada sea de tamaño (D,). Además, se está utilizando el principio de broadcasting, que es un mecanismo de Numpy para efectuar operaciones vectorizadas entre arreglos sin copiar los datos manualmente.

Explicación del código

Para la solución, se reciben como parámetros una matriz X de tamaño (N, D) y dos vectores: un vector b de tamaño (D, ) y un vector w de tamaño (N, ). Dado que los datos pueden ingresarse inicialmente como listas de Python, primero se convierten a arreglos de Numpy mediante np.array(), con el objetivo de desarrollar las operaciones vectorizadas.

Una vez convertidos, el vector b tiene el tamaño adecuado para sumarse a cada columna de la matriz X. Utilizando el mecanismo de broadcasting de Numpy, esta suma se realiza directamente, ya que Numpy expande automáticamente el vector b a todas las filas de la matriz sin necesidad de replicarlo.

Posteriormente, la forma original del vector w de tamnaño (N, ) no es compatible con una matriz de tamaño (N, D) para multiplicación elemento a elemento por filas. Por lo tanto, es necesario convertirlo a una forma (N, 1) utilizando reshape(-1,1). Finalmente, se realiza la operación (X + b) * w, obteniendo una nueva matriz de tamaño (N, D).

import numpy as np

def broadcast_ops(X, b, w):
    X = np.array(X)
    b = np.array(b)
    w = np.array(w)
    
    Y = (X + b) * w.reshape(-1,1)
    return Y

Ejercicio 2: multiplicación de matrices¶

Dadas dos matrices: A de tamaño (M, K) y B de tamaño (K, N), realice la operación del producto C = AB, de tamaño (M, N).

En este caso, se piden dos funciones, una que utilice estructuras for y sin vectorizar, utilizando tres blucles for anidados, y posteriormente una versión vectorizada utilizando funciones de Numpy.

Requerimientos
1- Una función que calcule el producto de dos matrices utilizando tres bulcles for anidados.
2- Otra función que utilice Numpy y optimice la operación anterior. Se pueden utilizar las operaciones "@" o np.dot de Numpy.
3- Para la primera función, no usar las operaciones "@" o np.dot de Numpy.
4- Para la segunda función, no usar bucles.

Explicación de la fundamentación matemática

A diferencia del ejercicio anterior, aquí sí se trabaja con la multiplicación matricial que se conoce de la teoría, donde cada elemento de la matriz resultante se obtiene multiplicando una fila de la primera matriz por una columna de la segunda y sumando los productos correspondientes entrada por entrada.

En cuanto a principios matemáticos, se utiliza la definición formal de la multiplicación de matrices, igual como aparece en la página de PaperCode, donde el elemento ubicado en la posición (i,j) de la matriz resultado se calcula como la suma de los productos entre los elementos de la fila i de A y la columna j de B. Esto se expresa matemáticamente como:

$$ C_{ij} = \sum_{k=0}^{K-1} A_{ik} B_{kj} $$

Es importante mencionar que con tal de que esta operación sea posible, las dimensiones internas deben coincidir. Es decir, si A tiene tamaño (M,K), entonces B debe tener tamaño (K,N). Esto garantiza que cada fila de la primera matriz tenga la misma cantidad de elementos que cada columna de la segunda matriz, y se devuelve una matriz de tamaño (M, N).

Explicación del código

Para la solución, se utilizó el "esqueleto" del código que se encuentra en el ejercicio 2 de el módulo de PaperCode, así, se reciben como parámetros dos matrices A y B, que pueden ingresarse inicialmente como arreglos de Numpy de dos dimensiones. Primero se obtienen sus dimensiones mediante .shape, identificando el número de filas y columnas de cada una, y verificando que las dimensiones internas sean compatibles previo a realizar la operación.

En la versión manual (matmul_naive), se inicializa una matriz de ceros de tamaño (M,N) para almacenar el resultado. Luego se utilizan tres ciclos anidados: el primero recorre las filas de A, el segundo las columnas de B, y el tercero realiza la suma acumulada de los productos correspondientes entre fila y columna para construir cada entrada de la matriz resultado.

Posteriormente, en la versión vectorizada (matmul_vectorized), se utiliza la operación optimizada de Numpy mediante np.dot(), que lleva a cabo el producto matricial.

import numpy as np

#A pesar de que en esta versión no se van a utilizar las funciones de Numpy vectorizadas, si se utiliza la función shape que devuelve el tamaño de la matriz, es por esto que primero en ambos casos es necesario convertir las matrices a objetos procesables por Numpy como los arreglos. 

def matmul_naive(A: np.ndarray, B: np.ndarray) -> np.ndarray:
    A = np.array(A)
    B = np.array(B)

    M, K = A.shape
    K2, N = B.shape
    assert K == K2 #Verifica que sea verdadero, para ejecutar la multiplicación

    C = np.zeros((M, N))

    for i in range(M):
        for j in range(N):
            for k in range(K):
                C[i,j] += A[i,k] * B[k,j]
    return C

#Función vectorizada
def matmul_vectorized(A: np.ndarray, B: np.ndarray) -> np.ndarray:
    A = np.array(A)
    B = np.array(B)
    return np.dot(A,B)

     [7, 8]]

import numpy as np

def elementwise_ops(a, b):
    a = np.array(a)
    b = np.array(b)
    epsilon = 1e-8 # Para evitar dividir por cero
    if a.shape == b.shape: #Verficamos si el tamaño coincide
        dicc = {"add": a + b, "mul": a * b , "div": a / (b + epsilon)}
        return dicc
    else: 
        return "No se puede aplicar las operaciones, el tamaño no coincide."

Ejercicio 4: remodelación y transposición de tensores

Implemente la función reshape_and_transpose(x, B, C, H, W) que realiza una conversión de datos de tipo imagénes común, es decir, conjuntos de matrices con vectores como elementos. Toma como input un vector con todos los valores que tendrá el tensor, y como parámetros recibe las especificaciones de la cantidad de imágenes, de las filas y columnas de cada imágen y del tamaño del vector.

Se deben hacer dos operaciones:

  1. reshape: utlizando el formato canales primero para convertir el vector x en un tensor de 4 dimensiones.
  2. transpose: utilizando el formato canales último, de esta manera transponer las matrices cambiando el tamaño de cada vector.

Requerimientos

  1. Verificar que len(x) == B * C * H * W antes de remodelar.
  2. Utilice las operaciones reshape de Numpy y transpose.
  3. Conservar los valores de los datos (solo reorganizarlos, no modificarlos).
  4. Devuelve el tensor con la forma correcta (B, H, W, C).

Explicación de la fundamentación matemática

La función reshape sigue la regla matemática de trabajar en 4 ejes cuando se tienen 4 dimensiones en una figura. Cada parámetro de las imagénes, canales, altura, anchura representan los valores que toma cada eje y de esta forma moldea el vector inicial de x en una figura multidimensional. Cuando se aplica reshape, obtenemos una lista de los ejes, donde cada uno representa el índice 0, 1, 2, 3 de esa lista.

Para utilizar la función de transpose es importante recordar la transposición de una matriz, donde se intercambian el número de filas y columnas entre sí sin modificar los valores iniciales. La aplicación de este concepto en este ejercicio tiene un enfoque multidimensional en el que no solamente se intercambian las filas y las columnas sino también los canales, que es el tamaño que tiene el vector, el cual es un elemento de las matrices. De igual manera, este método se puede aplicar a más de 4 dimensiones.

Explicación del código

Primero, es importante cambiar el tipo de estructura de la variable x a un tipo compatible con Numpy con la función np.array(). Luego, se debe verificar que el largo de x coincida con el total de todos los elementos que habría al agrupar estos valores con las especificaciones de las cantidades de imágenes, la altura, el ancho y los canales. Si esto se cumple, se aplica la función de reshape para dimensionar el array de x en 4 dimensiones y de esta forma obtener 4 ejes.

Al obtener el valor de los índices de los ejes, se utiliza la función transpose para indicar qué índice se intercambia según lo que se desea, en este caso, las imágenes se mantienen igual por lo tanto el primer índice sigue siendo 0, y el caso de los demás ejes, se cambian por este orden: (2, 3, 1). Se puede concluir que se modifica x al formato de canales primero y luego se transforma en el formato de canales último. Este último formato es utilizado para una mejor visualización de los datos de un tensor.

import numpy as np
def reshape_and_transpose(x, B, C, H, W):
    x = np.array(x)
    if len(x) == B * C * H * W:
        x = x.reshape(B, C, H, W)
        #Se cambia el orden de los ejes para obtener el formato canales último
        x = x.transpose(0, 2, 3, 1)
    return x

import numpy as np
from typing import Dict, Union

def tensor_reductions(x: np.ndarray, axis: int) -> Dict[str, Union[np.ndarray, float]]:
    return {
    # Ingresamos suma, promedio, máximo y argmax
    "sum": np.sum(x, axis=axis),
    "mean": np.mean(x, axis=axis),
    "max": np.max(x, axis=axis),
    "argmax": np.argmax(x, axis=axis)
    }

Ejercicio 6: normas de vectores¶

Implemente una función "compute_norms(x)" que toma una matriz "x" de tamaño (N, D) representando una cantidad de N vectores de dimensión D que retorne un diccionario con dos normas fundamentales:

Se visualizó anteriormente la forma de las normas. La de Manhattan, tiene como trabajo principal medir la distancia entre los vectores dados a través de la suma de las diferencias absolutas, lo cual puede interpretarse como el hecho de moverse en una cuadrícula. Por otra parte, la distancia euclidiana es la más reconocida en la enseñanza general, su popularidad se debe a que es la forma más "directa" de conseguir la distancia entre dos puntos, su uso más común es el teorema de Pitágoras.

En cuanto a la norma euclidiana, se calcula siguiendo la fórmula especificada, acudiendo a las operaciones de Numpy. Finalmente, ambos resultados se retornan en un diccionario.

import numpy as np
def compute_norms(x: np.ndarray) -> Dict[str, np.ndarray]:
    # L1 usamos axis = 1 para recorrer por filas

Ejercicio 7: producto punto y producto cruz

Implemente una función denominada producto_vectores(a,b), donde a y b son matrices (N,3), que representan lotes de vectores tridimensionales. Retorne un diccionario con el producto punto y el producto cruz de los parámetros.

Requerimientos
1- Calcule los productos de manera independiente para cada par de vectores en el lote.
2- Use las operaciones vectorizadas de Numpy.
3- Para el producto punto: use np.sum con un parámetro axis o np.einsum.
4- Para el producto cruz: implemente la fórmula componente por componente usando operaciones vectorizadas.
5- Maneje los lotes eficientemente, evite los bucles explícitos.

Explicación de la fundamentación matemática

Intuitivamente, el producto punto brinda una idea sobre qué tanto apuntan dos vectores en la misma dirección y para cada par de vectores tridimensionales se calcula con la siguiente fórmula: $$ a ⋅ b = \sum_{i=1}^{3} a_{i} b_{i} = a_{x}b_{x} + a_{y}b_{y} + a_{z}b_{z} $$ En palabras simples, se multiplican las entradas situadas en las mismas posiciones y se suma el resultado de estas multiplicaciones, de manera que se retorna un escalar. Es importante destacar que, con el objetivo de que el producto punto esté definido, es necesario que ambos vectores posean la misma dimensión.
Por otro lado, el producto cruz de dos vectores tridimensionales se calcula haciendo uso de la fórmula: $$ a × b = (a_{y}b_{z} - a_{z}b_{y}, a_{z}b_{x} - a_{x}b_{z}, a_{x}b_{y} - a_{y}b_{x}) $$ Es preciso señalar que el producto cruz está definido solo para vectores de tres dimensiones. El resultado corresponde a un vector tridimensional, que además es perpendicular a los vectores sobre los cuales se aplicó la fórmula.

Explicación del código

En primera instancia, se convierten las matrices del parámetro a un arreglo de Numpy, con el objetivo de efectuar óptimamente las operaciones vectorizadas. Después, se procede a calcular el producto punto siguiendo las indicaciones de los requerimientos: se emplea np.sum con el parámetro axis=1, el cual indica que se debe sumar sobre las filas del arreglo; note que el arreglo sobre el cual se está sumando corresponde a la multiplicación de a y b, en vista de que anteriormente se definieron como arreglos de Numpy, dicha multiplicación actúa sobre las entradas en las mismas posiciones de los vectores y retorna un arreglo con el resultado.
Por otra parte, para calcular el producto cruz componente por componente, se acude al slicing siguiendo la fórmula planteada. Cada componente se calcula por separado y representa un arreglo. Finalmente, los tres arreglos se unifican en un único arreglo de Numpy, organizándose como columnas.

import numpy as np

def producto_vectores (a,b):
    #Se convierten las matrices en arreglos de Numpy para efectuar las operaciones vectorizadas. 
    a = np.asarray(a)
    b = np.asarray(b)
    #Se calcula el producto punto usando np.sum, con parametro axis=1 para que se sumen las filas. 
    producto_punto = np.sum(a * b, axis=1)
    #Se calcula el producto cruz, componente por componente, de manera vectorizada.
    componente_x = a[:, 1] * b[:, 2] - a[:, 2] * b[:, 1] #ay*bz - az*by
    componente_y = a[:, 2] * b[:, 0] - a[:, 0] * b[:, 2] #az*bx - ax*bz
    componente_z = a[:, 0] * b[:, 1] - a[:, 1] * b[:, 0] #ax*by - ay*bx
    #Se agrupan los vectores en un array de Numpy, donde cada vector representa una columna. 
    producto_cruz = np.column_stack((componente_x, componente_y, componente_z))

    return {"Producto punto": producto_punto, "Producto cruz": producto_cruz}

#Se prueba el ejemplo.
print(producto_vectores([[1, 0, 0]], [[0, 1, 0]]))

{'Producto punto': array([0]), 'Producto cruz': array([[0, 0, 1]])}

Ejercicio 8: fundamentos de la sumatoria de Einstein (einsum)¶

Implemente la función "einsum_ops(A, B)" donde A es una matriz de tamaño (N, D) y B es una matriz de tamaño (D, M). Retorne un diccionario con cinco operaciones fundamentales, todas implementando "np.einsum". Las operaciones se detallan a continuación:

1- transpose: transponer A de tamaño (N, D) a (D, N).
2- sum: sumar todos los elementos de A (resultado escalar).
3- row_sum: suma de A sobre las filas (reducir columnas), el resultado debe ser un vector de tamaño (N, ).
4- col_sum: suma de A sobre las columnas (reducir filas), el resultado es un vector de tamaño (D, ).
5- mat_mul: producto matricial AB de tamaño (N, M).

Requerimientos
1- Usar "np.einsum" para todas las operaciones (no utilizar las funciones equivalentes de Numpy).
2- Entender la notación de los índices y cómo funciona la sumatoria.
3- Manejar el tamaño correcto de matrices y vectores para cada operación.
4- Retornar los resultados en el formato de diccionario especificado.

Explicación de la fundamentación matemática¶

En este ejercicio se trabaja con la notación de suma de Einstein, que es una forma compacta de representar operaciones matriciales y de tensores mediante índices. La idea principal es que cada índice representa una dimensión de la matriz, y cuando un índice aparece repetido en una expresión, se interpreta automáticamente como una suma sobre esa dimensión.

En las operaciones de suma y transposición, la notación permite indicar directamente qué índices se conservan y cuáles se eliminan. Por ejemplo, en la transposición de una matriz A, la expresión "ij->ji" indica que el índice de fila i y el de columna j intercambian su posición, generando una nueva matriz donde filas y columnas se invierten. Para la suma total "ij->", ambos índices desaparecen, lo que implica que todos los elementos de la matriz se suman en un único valor escalar.

Para las sumas parciales, la lógica es similar: en "ij->i" se conserva únicamente el índice de fila, por lo que se suman todos los elementos de cada fila sobre las columnas; mientras que en "ij->j" se conserva el índice de columna, sumando sobre las filas. Matemáticamente, en notación de sumatoria, esto corresponde a:

$$ R_i = \sum_{j} A_{ij} $$

$$ C_j = \sum_{i} A_{ij} $$ ​

En el caso del producto matricial, se utiliza la notación "ik,kj->ij", donde el índice k aparece repetido en ambas matrices y por lo tanto se suma automáticamente. Esto corresponde exactamente a la definición formal de multiplicación de matrices, donde cada elemento de la matriz resultado se obtiene multiplicando una fila de la primera matriz por una columna de la segunda y sumando los productos correspondientes:

$$ C_{ij} = \sum_{k=0}^{D-1} A_{ik} B_{kj} $$

Es importante mencionar que para que esta operación sea posible, las dimensiones internas deben coincidir. Es decir, si A tiene tamaño (N,D), entonces B debe tener tamaño (D,M). Esto garantiza que la dimensión compartida D pueda contraerse correctamente, obteniéndose una matriz resultado de tamaño (N,M).

Explicación del código

Para la solución, se define una función einsum_ops(A, B) que recibe como parámetros dos matrices A y B, ingresadas directamente como arreglos bidimensionales de Numpy. La matriz A tiene dimensión (N,D) y la matriz B dimensión (D,M), de manera que ambas sean compatibles para realizar el producto matricial en la última operación del diccionario.

Dentro de la función, cada operación se implementa utilizando np.einsum(), que funciona como se explicó en la teoría anterior. Primero, para la transpuesta de A, se utiliza "ij->ji", donde se intercambian los índices de filas y columnas, obteniendo una matriz de dimensión (D,N). Luego, para calcular la suma total de los elementos de A, se emplea "ij->", indicando que todos los índices desaparecen y por tanto todos los valores se suman en un único escalar.

Posteriormente, para obtener la suma por filas se utiliza "ij->i", conservando únicamente el índice de fila y sumando sobre las columnas, mientras que para la suma por columnas se utiliza "ij->j", conservando el índice de columna y sumando sobre las filas. Estas operaciones generan vectores unidimensionales con las sumas correspondientes.

Finalmente, para el producto matricial entre A y B, se utiliza "ik,kj->ij", donde el índice k representa la dimensión compartida entre ambas matrices y es el que se contrae automáticamente en la suma. De esta forma se obtiene una matriz resultado de dimensión (N,M), equivalente al producto matricial convencional realizado con np.dot().

Ejercicio 10: gradiente de una suma

Implemente la función grad_sum(grad_y, x_forma), la cual: tome grad_y, un escalar que representa ∂L/∂y; luego tome x_forma, una tupla que representa la forma original de la entrada x y finalmente retorne un tensor de gradiente de la forma x_forma, donde cada elemento iguale a grad_y.

Requerimientos
1- Retorne un tensor de la forma x_shape, relleno con grad_y.
2- Use las operaciones de Numpy, por ejemplo, np.full o np.ones * grad_y.
3- Maneje cualquier forma, no solo vectores de una dimensión.
4- Asegúrese de que el tipo de dato de la salida sea igual al tipo de dato de grad_y.

Explicación de la fundamentación matemática¶

En primera instancia, se define la siguiente operación: $$ y = sum(x) = \sum_{i=1}^{N} x_{i} $$ De la operación anterior, se tiene que x representa un vector de tamaño (N, ), el cual contiene N elementos. Por otro lado, y es un escalar que representa el resultado de la suma de todos los elementos de x.
Intuitivamente, se tiene que el gradiente corresponde a una tasa de cambio: la tasa de cambio de una función escalar respecto a variables determinadas, es decir, indica la dirección y la magnitud en la que cambia una función de acuerdo a ciertas variables. Así, al definir grad_y como ∂L/∂y donde L es la pérdida, se está midiendo qué tanto cambia la función de pérdida según y.
Note que se busca calcular ∂L/∂x, además recuerde que x es un vector formado por entradas xi; empleando la regla de la cadena y descomponiendo ∂L/∂x en cada ∂L/∂xi se tiene lo siguiente: $$ \frac{∂L}{∂x_{i}} = \frac{∂L}{∂y} ⋅ \frac{∂y}{∂x_{i}} $$ Se sabe que grad_y = ∂L/∂y, por lo que el enfoque recae sobre ∂y/∂xi. De nuevo, ∂y/∂xi representa qué tanto cambia la función y de acuerdo a xi. Dado que la función y está conformada por un único elemento xi (general), con el subíndice i que se encuentra entre 1 y N, derivando parcialmente se obtiene: $$ \frac{∂y}{∂x_{i}} = \frac{\sum_{j=1}^{N} x_{j}}{∂xi} = 1; 1 \leq i \leq N $$ Finalmente: $$ \frac{∂L}{∂x_{i}} = \frac{∂L}{∂y} ⋅ \frac{∂y}{∂x_{i}} = grad_y ⋅ 1 = grad_y $$ ​Dado que xi se tomó como un elemento general, visualice que el procedimiento detallado antes se repite para todos los elementos de x, de manera que ∂L/∂x corresponde a un vector del mismo tamaño de x, cuyas entradas son iguales al gradiente de la suma, grad_y.

Explicación del código

Siguiendo las instrucciones detalladas en los requerimientos, se utiliza la operación de Numpy denominada np.full. Dicha operación se encarga de crear un arreglo de Numpy recibiendo los siguientes parámetros: np.full(tamaño, valor_rellenar, dtype = tipo_de_dato). El parámetro tamaño indica precisamente el tamaño del arreglo que se creará, después, valor_rellenar representa el valor constante con el que se rellenarán todas las posiciones del arreglo, finalmente, tipo_de_dato corresponde al tipo de dato con el que se rellenarán las posiciones. Note que, con el objetivo de cumplir con el requerimiento que establece que el tipo de dato de la salida debe coincidir con el tipo de dato de grad_y, se emplea .dtype, un atributo de los arreglos de Numpy que describe el tipo de dato de los elementos que contiene el arreglo. Al convertir grad_y en un arreglo de Numpy y emplear sobre él .dtype, se obtiene el tipo de dato de grad_y.
En vista de que el gradiente se "divide" equitativamente: si la suma contribuyó con g a la pérdida, entonces cada elemento que contribuyó a la suma recibe un gradiente de g, y además el parámetro x_forma representa el tamaño original de la entrada x, se debe retornar un arreglo de tamaño x_forma cuyas entradas sean iguales al gradiente de la suma, en este caso, grad_y.

Dado lo anterior para hacer los cálculos correctos se utiliza la derivación, la regla de la cadena y el cálculo de matrices, tal que para el elemento Aik se necesita ∂L/∂Aik: $$ \frac{\partial L}{\partial A_{ik}} = \sum_{j=1}^{N} \frac{\partial L}{\partial C_{ij}} \cdot \frac{\partial C_{ij}}{\partial A_{ik}} $$ Dado que: $$ C_{ij} = \sum_{k'} A_{ik'} B_{k'j}, $$ Se obtiene finalmente: $$ \frac{\partial L}{\partial A_{ik}} = \sum_{j=1}^{N} G_{ij} B_{kj} = \sum_{j=1}^{N} G_{ij} (B^T)_{jk} $$ La vista en forma matricial queda: $$ \frac{\partial L}{\partial A} = G B^T $$ Pasa un caso similar con respecto a B, el cual concluye con la fórmula: $$ \frac{\partial L}{\partial B} = A^T G $$ Es importante verificar cada paso del desarrollo y tener especial cuidado al tratar las dimensiones de las matrices con las que se está trabajando.

Dada la explicación anterior se obtuvieron las fórmulas finales a utilizar, se calcula primero el gradiente de A, a través del uso "@" y np.transpose(); el @ se encarga de efectuar el producto punto y el transpose calcula la transpuesta requerida. def grad_matmul(grad_C: np.ndarray, A: np.ndarray, B: np.ndarray) -> Dict[str, np.ndarray]:
    # Confirmamos que todos sean arreglos 
    grad_C = np.array(grad_C)
    A = np.array(A)
    B = np.array(B)

    grad_A = grad_C @ np.transpose(B)
    grad_B = np.transpose(A) @ grad_C

        "grad_B": grad_B
    }

Ejercicio 12: codificación One-Hot¶

Implemente una función llamada one_hot_encode(indices, num_classes) que transforme un vector de etiquetas enteras en una representación de matriz binaria. La función debe identificar las categorías únicas, construir una matriz de dimensiones N×C (donde N es el número de muestras y C el número de clases) e insertar un 1.0 en la columna correspondiente al índice de cada muestra, manteniendo el resto de la fila en 0.0.

Requerimientos
1- Maneje casos borde: arreglos vacíos, una sola muestra, una sola clase.
2- Asegúrese de que todos los índices sean válidos ($0 \le \text{índice} < C$).
3- Devuelva un arreglo de tipo flotante (dtype=float32 o float64) para estabilidad numérica.
4- Utilice operaciones vectorizadas (evite bucles explícitos).

Explicación de la fundamentación matemática¶

La idea principal de este método es convertir categorías en posiciones dentro de un espacio vectorial. El problema a resolver es que números como 0, 1 o 2, cuando representan clases, no tienen un significado de magnitud; o sea, que una clase sea “2” no significa que sea mayor que “0”, solo que es distinta. Para evitar que un modelo matemático interprete mal esto, cada categoría se transforma en un vector. Es decir, si hay C clases en total, cada dato se convierte en una fila de una matriz donde solo una posición tiene valor 1.0 y todas las demás posiciones tienen valor 0.0. En términos simples, se crea un espacio con tantas dimensiones como categorías haya, para cada dato se busca la posición que corresponde a su clase y en esa posición se coloca un 1.0, y en las demás 0.0.

Matemáticamente, esto se puede expresar como una función indicadora, donde cada componente del vector está definido como:

$$ one\_hot[j,k] = \begin{cases} 1 & k = i_j \\ 0 & \text{en otro caso} \end{cases} $$

Esto significa que el valor es 1 únicamente cuando la posición coincide con la clase correcta, y 0 en cualquier otro caso. Esto es importante porque convierte cada etiqueta en una distribución de probabilidad perfecta. Es decir, al tener un “1” en la clase correcta y “0” en las demás, se está diciendo que hay 100% de certeza de que el dato pertenece a esa categoría.

Explicación del código¶

Este código lo que hace es convertir una lista de clases en una matriz, donde cada clase se representa como un vector. Primero, toma la lista de índices y la convierte en un arreglo de Numpy para poder trabajar de forma vectorizada. Luego, obtiene cuántos datos hay (N). Si la lista está vacía, devuelve directamente una matriz vacía con el número correcto de columnas (una por cada clase) para evitar errores. Después, verifica que todos los valores de índices estén dentro del rango válido, es decir, que no sean negativos ni mayores o iguales al número de clases, evitando así acceder a posiciones que no existen en la matriz.
A continuación, crea una matriz llena de ceros de tamaño (N, C), donde N es la cantidad de datos y C es el número de clases; esa matriz es donde se construye la representación buscada. En lugar de usar ciclos, se utiliza indexación avanzada de Numpy: np.arange(N) indica las filas e índices, además señala en qué columna va el 1 en cada fila. Con esto, Numpy coloca un 1.0 en la posición correcta de cada fila y deja el resto en 0.0. Al final, devuelve la matriz resultante, donde cada fila representa un dato y tiene un 1 en la posición de su clase y 0 en todas las demás.

import numpy as np

def one_hot_encode(indices: np.ndarray, num_classes: int) -> np.ndarray:
    
    # convertir a array de numpy
    indices = np.array(indices, dtype=np.int64)
    N = indices.shape[0]

    # caso de un array vacío
    if N == 0:
        return np.zeros((0, num_classes), dtype=np.float64)

    # si hay índices inválidos devuelve matriz de ceros
    if np.any(indices < 0) or np.any(indices >= num_classes):
        return np.zeros((N, num_classes), dtype=np.float64)

    # crear matriz de ceros
    result = np.zeros((N, num_classes), dtype=np.float64)

    # asignación con indexación avanzada utilizando sugerencia
    result[np.arange(N), indices] = 1.0

    return result

Ejercicio 13: implementación de softmax

Implemente una función softmax(x), que convierta logits en probabilidades.
La entrada x corresponde a una matriz de la forma (N,C), donde N es el tamaño del lote (el número de muestras) y C es el número de clases; cada fila contiene logits (puntuaciones brutas) para una muestra. La salida debe ser una matriz de la forma (N,C), donde cada fila representa una distribución de probabilidad válida: todos los valores son mayores o iguales a cero, cada fila suma uno y logits más altos corresponden a probabilidades más altas.
En cuanto al cálculo, se debe calcular el softmax de manera independiente para cada fila. Además, para cada muestra convierta sus C logits en C probabilidades.

Requerimientos
1- Implemente la versión numéricamente estable (restar el máximo antes de exponenciar).
2- Calcule el softmax de manera independiente para cada fila.
3- Retorne probabilidades de la forma (N,C), donde cada fila suma 1.
4- Maneje casos límite: todos los ceros, los valores muy grandes y los valores muy pequeños.
5- Use las operaciones de Numpy para cálculos vectorizados.

Explicación de la fundamentación matemática

La función softmax es un componente matemático que se encarga de convertir un vector de números reales, cuyas entradas se denominan logits, en una distribución de probabilidad válida: los valores de las probabilidades se encuentran en [0,1] y las filas suman un total de 1. A continuación, se detalla la fórmula aplicada para calcular el softmax: $$ softmax(x)_{i} = \frac{e^{x_{i} - max(x)}}{\sum_{j=1}^{C} e^{x_{j} - max(x)}} $$ La resta situada en el exponente, de cada elemento respecto al máximo valor (logit) de la fila sobre la que se está trabajando, es un agregado a la fórmula cuyo objetivo consiste en evitar el desbordamiento numérico. Lo anterior es factible debido a la propiedad de la invarianza: agregar una constante a todos los logits no cambia las probabilidades. De esta forma, se asegura la estabilidad numérica. Asimismo, la exponenciación de euler hace que todos los valores sean positivos.
Por otro lado, la constante de normalización se encarga de asegurar que las probabilidades de las filas sumen 1.
Todo esto en conjunto, genera la construcción de una distribución de probabilidad válida.

Explicación del código

En primera instancia, se convierte la matriz en un arreglo de Numpy con el objetivo de aplicar óptimamente las operaciones vectorizadas necesarias. Después, se calcula el valor máximo (logit) de cada fila y se calculan los exponentes de cada entrada. Para realizar la resta en los exponentes, es preciso actualizar la forma del vector de los valores máximos, de manera que sea un vector con una única columna, cuyas filas representan el valor máximo de cada fila de la matriz, así por ejemplo, cada entrada de la primera fila de la matriz se restará con el valor de la primera fila del vector de los valores máximos, el cual corresponde al máximo valor de la primera fila de la matriz; este proceso se repite con cada una de las filas.
Seguidamente, se exponencian las entradas haciendo uso de la operación np.exp, la cual se encarga de exponenciar euler a los valores del parámetro, que en este caso corresponde a un arreglo de Numpy, por lo que se mantiene la estructura de la matriz.
Luego, se procede a calcular las constantes de normalización de cada fila, que corresponden a la suma de los elementos exponenciados de la fila correspondiente. Por último, similar al caso de la resta, para efectuar la división vectorizada de las entradas de una fila con la constante de normalización asociada a dicha fila, se necesita que el vector de las constantes de normalización posea una única columna, por lo que se recurre a la función de .reshape().

import numpy as np
def softmax (x): 
    #Se convierte la matriz a un arreglo de Numpy para aplicar operaciones vectorizadas.
    x = np.asarray(x)
    #Se extraen los valores máximos de cada fila.
    max_filas = np.max(x, axis=1)
    #Se calculan los exponente de cada fila. Se convierte el vector max en un vector de una única columna. 
    exponentes = x - max_filas.reshape(-1,1)
    #Se calculan las exponenciales con la operacion np.exp.
    exponenciales = np.exp(exponentes)
    #Se suman las exponenciales por filas para obtener la constantes de normalizacion.
    constantes_normalizacion = np.sum(exponenciales, axis=1)
    #Se normaliza y se contruye el softmax. De nuevo, se convierte el vector de las constantes de normalizacion a un vector columna. 
    softmax = exponenciales / constantes_normalizacion.reshape(-1, 1)
    
    return softmax

Ejercicio 14: pérdida de entropía cruzada (manual)¶

Implemente una función llamada cross_entropy_loss(probs, targets) que calcule el promedio de la pérdida de entropía cruzada para un lote de muestras. La función debe recibir una matriz de probabilidades de forma (N,C) y un vector de etiquetas reales, para luego extraer la probabilidad asignada a la clase correcta de cada muestra mediante indexación avanzada. Finalmente, debe calcular el logaritmo negativo de estas probabilidades, aplicando un factor de estabilidad numérica (ϵ=1e−9), y retornar el promedio escalar de estos valores para todo el lote.

Requerimientos
1- Utilice la indexación avanzada para seleccionar eficientemente las probabilidades correctas.
2- Añada un épsilon (ϵ=1e−9) dentro del logaritmo para la estabilidad numérica.
3- Devuelva un escalar (pérdida promedio sobre el lote).
4- Maneje casos borde: predicciones perfectas, predicciones muy erróneas.
5- Utilice operaciones vectorizadas de Numpy (evite bucles explícitos).

Explicación de la fundamentacion matemática¶

La entropía cruzada mide qué tan bien le está yendo al modelo al asignarle probabilidad a la respuesta correcta. Si le da una probabilidad alta, el error es pequeño; si le da una probabilidad baja, el error es grande.

En un problema de clasificación, el modelo produce una distribución de probabilidades:

$$ p = [p_1, p_2, \dots, p_C] $$

Por otro lado, la respuesta real se representa como un vector one-hot, donde toda la probabilidad está en la clase correcta. Para una sola muestra cuya clase verdadera es $y_i$, la entropía cruzada se define como:

$$ H(p, y_i) = -\log(p_{i, y_i}) $$

Esto significa que solo importa la probabilidad que el modelo le da a la clase correcta.

Cuando se tienen N muestras, se calcula el promedio de la pérdida como:

$$ Loss = -\frac{1}{N} \sum_{i=1}^{N} \log(p_{i,y_i} + \epsilon) $$

donde: - $p_{i,y_i}$ es la probabilidad asignada a la clase correcta en la muestra $i$. Por otro lado, $\epsilon$ es un valor pequeño (por ejemplo $10^{-9}$) que se agrega para evitar calcular $\log(0)$. Es importante resaltar que si $p_{i,y_i} = 1$, entonces $-\log(1) = 0$, lo que significa que no hay error. En cambio, si $p_{i,y_i}$ se acerca a 0, entonces $-\log(p_{i,y_i})$ crece hacia infinito, lo que implica un error muy grande.

Por eso, la entropía cruzada castiga fuertemente cuando el modelo se equivoca con mucha seguridad, y recompensa cuando acierta con alta probabilidad, ayudando a que el modelo mejore sus predicciones.

Explicación del código¶

La implementación está pensada para ser eficiente usando operaciones vectorizadas, lo que permite trabajar con muchos datos al mismo tiempo sin usar ciclos for, según la sugerencia que dio el módulo de evitar loops explícitos. Primero, se maneja la estabilidad numérica definiendo una constante eps = 1e-9. Esto se hace para evitar calcular log(0), ya que ese valor no está definido y causaría errores. Esto se incluye según la sugerencia que dio el módulo de agregar un épsilon dentro del logaritmo.
Luego, para obtener las probabilidades correctas, en lugar de recorrer la matriz fila por fila, se utiliza indexación avanzada con probs[np.arange(N), targets]. Esto permite extraer directamente, de cada fila, la probabilidad que corresponde a la clase real, según la sugerencia que dio el módulo de usar indexación avanzada. Después, se calcula la pérdida aplicando np.log() a esas probabilidades y luego np.mean() para obtener el promedio del logaritmo negativo, siguiendo la fórmula $-\frac{1}{N}\sum \log(p_{i,y_i})$ de forma vectorizada. Finalmente, la función devuelve un solo número (float) que representa la pérdida promedio del modelo en el conjunto de datos.

import numpy as np

def cross_entropy_loss(probs: np.ndarray, targets: np.ndarray) -> float:
    # Convertir a arrays de NumPy por si vienen como listas (ejemplo 2)
    probs = np.array(probs)
    targets = np.array(targets)

    # definimos un épsilon pequeño para evitar que el log(0) dé error
    eps = 1e-9
    N = probs.shape[0]

    # Indexación avanzada solicitada en los requerimientos para extraer la probabilidad de la clase correcta
    # Esto selecciona de cada fila i el valor en la columna indicada por targets[i]
    correct_probs = probs[np.arange(N), targets]

    # Cálculo de la pérdida usando el logaritmo negativo y aplicamos la fórmula del promedio
    loss = -np.mean(np.log(correct_probs + eps))

    return float(loss)

Ejercicio 15: estabilidad Numérica (Log-Sum-Exp)¶

Implemente una función llamada log_sum_exp(x) que calcule el logaritmo de la suma de exponenciales $\log(\sum e^{x_i})$ de forma numéricamente estable. La función debe recibir una matriz de forma (N,D) y aplicar el "truco del máximo" para evitar el desbordamiento (overflow) que ocurre al elevar números grandes a una potencia exponencial. La operación debe realizarse de forma independiente para cada fila, reduciendo la última dimensión para devolver un vector de tamaño (N).

Requerimientos
1- Implemente la versión numéricamente estable utilizando el truco de la resta del máximo.
2- Calcule log-sum-exp independientemente para cada fila (reduzca a lo largo de la última dimensión).
3- Devuelva un vector de forma (N,).
4- Maneje casos borde: todos ceros, valores muy grandes, valores muy pequeños.
5- Utilice operaciones de Numpy para computación vectorizada.
6- Asegure la estabilidad numérica para cualquier rango de entrada.

Explicación de la fundamentación matemática¶

Si se calcula directamente $\log(\sum e^{x_i})$, puede haber problemas de desbordamiento numérico cuando los valores de $x_i$ son muy grandes. Esto pasa porque la función exponencial crece demasiado rápido y puede superar el límite de los números que la computadora puede manejar. Para evitar eso, se usa el truco de log-sum-exp, que se basa en esta identidad: $$ \log\left(\sum_{i=1}^{D} e^{x_i}\right) = \max(x) + \log\left(\sum_{i=1}^{D} e^{x_i - \max(x)}\right) $$

La idea es restarle el valor máximo max(x) a todos los elementos antes de aplicar la exponencial. Así, el valor más grande se vuelve 0 y su exponencial es 1, y los demás quedan negativos, por lo que sus exponenciales están entre 0 y 1. Con esto se evita el desbordamiento. Al final, se vuelve a sumar el valor máximo fuera del logaritmo para que el resultado siga siendo equivalente al original.

Explicación del código¶

Para resolverlo, primero se convierte la entrada en un arreglo de Numpy. Luego, se obtiene el valor máximo de cada fila usando np.max(x, axis=1). Como ese resultado es un vector, se usa .reshape(-1, 1) para convertirlo en una columna y poder restarlo fácilmente a toda la matriz fila por fila. Después, se calcula la exponencial de los valores ajustados (exp_ajustada) y se suman por cada fila. Finalmente, se aplica el logaritmo a esas sumas y se le vuelve a agregar el valor máximo que se había restado al inicio. Así se obtiene el resultado correcto, pero de forma numéricamente estable, evitando problemas con valores infinitos.

import numpy as np

def log_sum_exp(x):
    x = np.array(x)
    
    # se obtiene el máximo de cada fila 
    max_filas = np.max(x, axis=1)
    
    # se ajusta x restándole el máximo de su respectiva fila
    # se usa reshape(-1, 1) para que sea una columna y pueda restarse a la matriz
    x_ajustado = x - max_filas.reshape(-1, 1)
    
    # exponencial de los valores ajustados
    exp_ajustada = np.exp(x_ajustado)
    
    # suma de exponenciales por fila
    suma_exp = np.sum(exp_ajustada, axis=1)
    
    # logaritmo de la suma + el máximo que restamos al inicio
    resultado = np.log(suma_exp) + max_filas
    
    return resultado
